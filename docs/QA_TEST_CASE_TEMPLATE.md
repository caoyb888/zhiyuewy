## 3. 通用测试检查项

每个 Story 测试时必须逐项确认：

### 3.1 接口规范检查

- [ ] 新增接口路径以 `/resi/` 开头，不使用 `/payment/` 或 `/baseconfig/` 前缀
- [ ] Controller 继承 `BaseController`，返回 `AjaxResult.success()` 或 `AjaxResult.error()`
- [ ] 分页接口使用 `startPage()` + `getDataTable()`
- [ ] 核心写操作（收款/退款/冲红/生成应收）有 `@Transactional` 注解
- [ ] 核心写操作有并发保护（FOR UPDATE 或 Redis 锁）
- [ ] 涉及金额的字段使用 `BigDecimal`，无 `double` / `float`
- [ ] 查询接口有 `projectId` 校验或 `@ResiProjectScope` 隔离

### 3.2 日志与审计检查

- [ ] 新增/修改/删除操作在 `sys_oper_log` 有记录
- [ ] 操作日志含操作人、时间、IP、请求参数（脱敏后）
- [ ] 无 `System.out.println` 输出

### 3.3 安全与数据检查

- [ ] 身份证号在数据库中为密文（AES-256）
- [ ] 手机号在前端展示为脱敏格式（138****8888）
- [ ] SQL 使用 `#{}` 占位符，无 `${}` 拼接
- [ ] 无硬编码密钥、密码、AppSecret

### 3.4 前端规范检查

- [ ] 所有 API 调用通过 `src/api/resi/*.js` 封装，组件中无直接 `axios` 调用
- [ ] 金额显示使用千分位 + 保留2位小数
- [ ] 收银台收款按钮有 `disabled + loading` 双重防重复点击
- [ ] 表格分页参数与后端一致（`pageNum` / `pageSize`）

### 3.5 数据库检查

- [ ] 新增表以 `resi_` 前缀，字符集 `utf8mb4`
- [ ] 档案类表含 `enabled_mark` 软删除字段
- [ ] 流水类表不含 `@TableLogic`，禁止物理删除
- [ ] 金额字段使用 `DECIMAL`，无 `FLOAT` / `DOUBLE`
- [ ] 外键字段有索引，业务唯一键已建立

## 5. 缺陷等级定义

| 等级 | 标识 | 定义 | 处理要求 |
|---|---|---|---|
| **P0** | 🔴 阻塞 | 系统崩溃、数据错误、权限穿透、核心流程中断 | 必须立即修复，禁止带至下阶段 |
| **P1** | 🟠 严重 | 功能不符合需求、关键路径异常、并发安全问题 | Sprint 内必须修复 |
| **P2** | 🟡 一般 | 界面显示问题、非核心功能异常、性能轻微下降 | 下一 Sprint 修复或排期 |
| **P3** | 🟢 轻微 | 文案错误、样式偏差、建议优化 | 可记录遗留，不阻塞上线 |
